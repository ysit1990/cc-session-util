# Claude Code 会话迁移工具

[中文](README.md) | [English](README.en.md)

一个轻量的桌面工具（Tkinter），用于在项目路径变更后迁移 Claude Code 会话目录，避免会话丢失。

> 当前版本：`v1.0.0`

## 功能特性

- 会话迁移：支持**复制**（保留原会话）或**移动**（删除原会话）
- 自动匹配：根据“原项目目录”匹配 Claude 会话目录名
- 手动刷新：支持手动刷新当前路径与会话目录的匹配结果
- 会话删除：可删除匹配到的会话目录（含二次确认）
- 配置直达：一键打开 Claude 配置目录（`~/.claude/projects`）
- 可视化列表：展示现有会话目录及推测项目路径，点击可回填原路径

## 适用场景

- 你将项目目录从 `D:\old\repo` 移动到 `D:\new\repo`
- Claude Code 因路径变化找不到旧会话
- 你希望将旧路径会话映射到新路径并继续使用

## 工作原理（简述）

工具会将项目路径编码为 Claude 会话目录名格式，并在 `~/.claude/projects` 中执行目录复制、移动或删除。

当前编码规则：

- Windows 盘符后使用 `--`（例如 `d:/work/demo` -> `d--work-demo`）
- 路径分隔符 `/` 映射为 `-`
- 原始路径中的 `_` 也按 `-` 处理
- 匹配时兼容部分旧格式（如盘符后单 `-`）

## 环境要求

- Python 3.8+
- 标准库依赖（无需额外安装第三方包）：`tkinter`、`pathlib`、`shutil` 等

## 快速开始

### 1) 直接运行源码

```bash
python main.py
```

### 2) 图形界面使用步骤

1. 选择或填写“原项目目录”
2. 选择或填写“新项目目录”
3. 选择模式：`复制(保留)` 或 `移动(删除原)`
4. 点击 `执行迁移`

补充操作：

- 点击 `刷新`：重新匹配当前原项目路径对应的会话目录
- 点击 `删除`：删除当前原项目路径匹配到的会话目录
- 点击 `打开配置目录`：打开 `~/.claude/projects`

## 构建 EXE（Windows）

项目已提供 `build.bat`：

```bat
build.bat
```

脚本会自动：

1. 检查 Python
2. 检查并安装 PyInstaller（如未安装）
3. 打包为单文件窗口程序

构建产物：

- `dist/cc-session-util.exe`

## 注意事项

- 删除会话目录是不可恢复操作，请确认后执行
- “推测的项目路径”是展示用途，因编码信息损失可能不完全精确
- 若匹配失败，请优先使用 `刷新`，并检查原项目路径是否与历史路径一致

## 常见问题（FAQ）

### Q1：为什么列表里路径看起来不完全对？

会话目录名编码时，`_` 与路径分隔符都可能映射为 `-`，解码时存在信息损失，因此展示路径是“推测值”。

### Q2：迁移时报“目标会话目录已存在”怎么办？

说明新路径对应的会话目录已存在。你可以选择覆盖，或先手动备份目标目录后再迁移。

### Q3：如何确认当前原路径匹配到了哪个会话目录？

查看界面中的“匹配会话目录名”，或点击 `刷新` 后查看提示信息。
